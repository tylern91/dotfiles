# tylern91/dotfiles

macOS dotfiles managed with GNU Stow, built around **Ghostty + Zsh + Antidote**.

## Project Overview

This is a personal dotfiles repository for macOS (Apple Silicon). Configuration files are organized into stow packages and symlinked into `$HOME` using GNU Stow.

## Architecture

```bash
dotfiles/
├── common/                    # Cross-platform configs → stowed to $HOME
│   ├── .zshrc                 # Zsh configuration (Antidote, cached evals, lazy NVM)
│   ├── .zsh_plugins.txt       # Antidote plugin list
│   └── .config/git/config     # Git configuration
├── macos/                     # macOS-specific configs → stowed to $HOME
│   └── .config/ghostty/config # Ghostty terminal configuration
├── .claude/                   # Claude Code configuration
├── Brewfile                   # Homebrew dependencies
├── install.sh                 # Installer script
└── README.md
```

## Key Commands

```bash
# Full install (backup, brew, stow, antidote, fonts)
./install.sh install

# Individual commands
./install.sh brew         # Install Homebrew dependencies
./install.sh antidote     # Regenerate Antidote static plugin bundle
./install.sh fonts        # Install Nerd Fonts
./install.sh backup       # Backup existing dotfiles
./install.sh uninstall    # Remove all symlinks
./install.sh restow       # Re-stow (uninstall + install)
./install.sh status       # Show current dotfiles status
```

## Architecture Patterns

### GNU Stow Packages

- `common/` — cross-platform configs, always stowed
- `macos/` — macOS-specific configs, stowed only on Darwin

### Zsh Performance Optimizations

- Antidote static bundles (compiled `.zsh_plugins.zsh` from `.zsh_plugins.txt`)
- Cached eval outputs in `~/.zsh/cache/` (pyenv, direnv, goenv, npm, ng)
- Poetry completions cached as fpath file (`~/.zsh/completions/_poetry`)
- Lazy-loaded NVM — only initializes on first `nvm` call
- Smart `compinit -C` — full rescan at most once per day
- Hardcoded `BREW_PREFIX=/opt/homebrew` (Apple Silicon)

## Adding New Configurations

1. **Cross-platform tool**: Add to `common/.config/<tool>/`
2. **macOS-specific tool**: Add to `macos/.config/<tool>/`
3. **New Homebrew dependency**: Add to `Brewfile`
4. **New Zsh plugin**: Add to `common/.zsh_plugins.txt`, run `./install.sh antidote`
5. After adding files, run `stow -t "$HOME" -v --adopt common` (or `macos`)

## Development Rules

- Use conventional commits: `type(scope): description`
- Test changes with `./install.sh status` before committing
- Clear relevant cache after updating tools: `rm ~/.zsh/cache/<tool>.zsh`
- Regenerate Antidote bundle after editing plugin list: `./install.sh antidote`

## Testing & Profiling Zsh

### Antidote is synchronous — not async

Antidote sources a pre-compiled static `.zsh_plugins.zsh` file at startup. **There is no async/turbo mode** (unlike Zinit's `wait` keyword). Speed comes from one-time compilation — the bundle is only regenerated when `.zsh_plugins.txt` changes. `zsh-autosuggestions` has its own async fetch mode (`ZSH_AUTOSUGGEST_USE_ASYNC=1`), which is separate — it fetches suggestions in a background process without blocking the prompt.

### Measure startup time

```bash
# Single run
time zsh -i -c exit

# 5-run average (rule out warm-up noise)
for i in $(seq 5); do time zsh -i -c exit; done 2>&1

# Statistical benchmark
hyperfine "zsh -i -c exit"
```

### Profile which functions are slow (zprof)

```bash
# 1. Add at the very TOP of ~/.zshrc (before BREW_PREFIX):
zmodload zsh/zprof

# 2. Add at the very BOTTOM of ~/.zshrc:
zprof

# 3. Open a new shell — a sorted profile table is printed
# 4. Remove both lines when done
```

### Verify/regenerate the static bundle

```bash
# Bundle .zsh should be newer than .txt:
ls -la ~/.zsh_plugins.txt ~/.zsh_plugins.zsh

# Force regeneration:
./install.sh antidote

# Inspect what Antidote compiled:
head -60 ~/.zsh_plugins.zsh
```

### Cache invalidation

```bash
rm ~/.zsh/cache/pyenv.zsh           # regenerates on next shell start
rm ~/.zsh/cache/direnv.zsh
rm ~/.zsh/cache/goenv.zsh
rm ~/.zsh/cache/npm_completion.zsh
rm ~/.zsh/cache/ng_completion.zsh
rm ~/.zsh/cache/fzf.zsh
rm ~/.zsh/completions/_poetry        # poetry completions
rm -rf ~/.zsh/cache ~/.zsh/completions  # nuke all caches
```

## Don't Touch

- `~/.zsh/cache/` files are auto-generated — don't edit manually
- `.zsh_plugins.zsh` is generated by Antidote — only edit `.zsh_plugins.txt`
- Don't commit `.env`, API keys, credentials, or private keys
